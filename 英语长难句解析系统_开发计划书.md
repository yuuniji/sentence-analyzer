## 一、项目概述

### 1.1 项目目标

构建一套**完全本地运行**的英语长难句智能解析系统，以 Gemini API 为核心推理引擎，自动将任意英语句子转化为符合 VitePress Tabs 插件规范的结构化解析文档，实现以下五大模块的自动生成：

- 完整翻译（意译 + 直译）
- 语义/意群拆分（分隔符 + 对照表格）
- 关键词汇与短语提取
- 重难点语法分析（主干 + 枝叶 + 逻辑难点）
- 导师小结

### 1.2 核心约束

| 约束项 | 规范 |
|--------|------|
| 输出格式 | 严格符合 VitePress Tabs 插件语法 |
| Emoji 禁用 | 输出内容中绝对不包含任何 Emoji |
| 排版风格 | 学术严谨风格，依赖加粗、缩进、留白 |
| 运行环境 | 本地化，不依赖外部服务（Gemini API 除外） |
| 语言角色 | 模拟资深英语语言学专家的分析视角 |

### 1.3 产出物形态

```
用户输入英文句子
       ↓
系统调用 Gemini API（携带结构化提示词）
       ↓
解析结构化 JSON 响应
       ↓
渲染为 VitePress Tabs Markdown 文本
       ↓
用户可直接复制粘贴至 VitePress 博客
```

```
Phase 2 · 提示词构造
│
├── 2.1 选择提示词模板（标准句 / 复杂句 / 段落模式）
├── 2.2 注入句子及清洗后的预处理结果
├── 2.3 注入格式规范约束
└── 2.4 注入角色设定与输出规则

Phase 3 · Gemini API 调用
│
├── 3.1 选择模型（gemini-3.1-pro-preview / gemini-2.5-flash）
├── 3.2 配置参数
│   ├── temperature: 0.3（保证稳定性）
│   ├── top_p: 0.9

├── 3.3 发起流式请求（generate_content with stream=True）
└── 3.4 实时推送 SSE 至前端

Phase 4 · 响应解析与格式化
│
├── 4.1 原始文本接收完毕
├── 4.2 结构验证
│   ├── 检测 <details> 包裹是否完整

├── 5.1 写入 SQLite（原文 + 解析结果 + 时间戳）
└── 5.2 更新历史列表
```

### 3.2 主干提取子工作流

虽然大模型输出中包含 `<details><summary>` 标签（主要用于后端格式兜底与博客导出兼容），但前端在渲染时会将其彻底剥离（平铺展示），直接呈现简洁的标签页（Tabs）：

```python
def extract_sentence_trunk(sentence: str) -> str:
    """
    策略：取句子前 80 个字符，在最近的逗号/分号处截断，
    追加省略号。若句子本身较短则直接使用原句。
    """
    if len(sentence) <= 100:
        return sentence
    
    trunk = sentence[:80]
    # 在最近的标点处断开
    for punct in [',', ';', ' and ', ' or ', ' but ']:
        idx = trunk.rfind(punct)
        if idx > 40:
            return trunk[:idx] + '...'
    
    return trunk[:75] + '...'
```

```

专注于英语句法分析、语法教学与中英翻译。

【身份约束】
- 你始终以严谨、耐心的语言学导师身份进行分析
- 你的解析具有学术权威性，同时保持可读性与亲和力

【绝对禁止项】（违反则输出无效）
- 禁止在任何位置使用 Emoji 表情符号

<summary>{{TRUNK_TEXT}}</summary>

::: tabs
== 完整翻译

**意译：** {{IDIOMATIC_TRANSLATION}}

**直译：** {{LITERAL_TRANSLATION}}（仅在意译与直译存在明显差异时提供）

== 语义/意群拆分

{{SEGMENTATION_WITH_SLASH}}

{{SEGMENT_TABLE_IN_MARKDOWN}}

== 关键词汇与短语

{{VOCABULARY_LIST}}

== 重难点与语法分析

**寻找主干：** {{CORE_STRUCTURE}}

**梳理枝叶：** {{BRANCH_ANALYSIS}}

**逻辑难点：** {{LOGIC_DIFFICULTY}}

{{SUMMARY_IN_1_TO_2_SENTENCES}}

:::

</details>

【各模块详细规范】

完整翻译模块：
- 意译须符合中文母语表达习惯，不得出现翻译腔
- 直译须逐字对应，可用于辅助理解句子结构
- 若意译已足够清晰，直译可省略

语义/意群拆分模块：
- 使用 / 作为意群分隔符，前后各留一个空格
- 表格必须包含"意群"和"中文含义"两列
- 使用标准 Markdown 表格语法（|---|---|）

关键词汇与短语模块：
- 格式：- **英文词汇** — *词性缩写*：中文释义。补充说明。
- 词性缩写规范：n./v./adj./adv./prep./conj./phr.
- 每条词汇注释须包含：当前语境释义 + 词源/同义词/搭配扩展

重难点与语法分析模块：
- 主干标注须使用代码格式或加粗突出核心成分
- 枝叶分析须逐一编号（①②③...）
- 每个语法现象须给出：名称 + 结构说明 + 还原示例（如适用）
- 逻辑难点须指出最易产生的误读及正确解读方法

导师小结模块：
- 严格控制在 1-2 句话
- 提炼本句最核心的记忆点或防坑要点
- 语气可适度亲切，但保持专业感
```

## 六、模块详细设计

### 6.1 提示词引擎（PromptEngine）

```python
# prompt_engine.py

from jinja2 import Environment, FileSystemLoader
from pathlib import Path
from dataclasses import dataclass
from typing import Optional


@dataclass
class AnalysisRequest:
    sentence: str
    trunk_text: str
    mode: str = "standard"          # standard | compact | paragraph
    context_sentences: Optional[str] = None
    custom_terms: Optional[dict] = None


class PromptEngine:
    """
    负责动态构造发送给 Gemini API 的提示词。
    系统提示词（System Prompt）固定加载，
    任务提示词（Task Prompt）根据请求参数动态渲染。
    """
    
    def __init__(self, templates_dir: str = "prompts/"):
        self.env = Environment(
            loader=FileSystemLoader(templates_dir),
            trim_blocks=True,
            lstrip_blocks=True
        )
        self._system_prompt = self._load_system_prompt()
    
    def _load_system_prompt(self) -> str:
        template = self.env.get_template("system_prompt.j2")
        return template.render()
    
    def build_task_prompt(self, request: AnalysisRequest) -> str:
        template_name = f"task_{request.mode}.j2"
        template = self.env.get_template(template_name)
        return template.render(
            sentence=request.sentence,
            trunk_text=request.trunk_text,
            context_sentences=request.context_sentences,
            custom_terms=request.custom_terms
        )
    
    @property
    def system_prompt(self) -> str:
        return self._system_prompt
```

### 6.2 Gemini 客户端封装（GeminiClient）

```python
# gemini_client.py

import google.generativeai as genai
from google.generativeai.types import GenerationConfig
from typing import AsyncGenerator
import asyncio


class GeminiClient:
    """
    封装 Gemini API 调用，支持流式和非流式两种模式。
    内置重试机制和错误处理。
    """
    
    MODEL_CONFIGS = {
        "standard": {
            "model": "gemini-2.5-flash",
            "temperature": 0.3,
            "top_p": 0.9,
            "max_output_tokens": 8192,
        },
        "pro": {
            "model": "gemini-3.1-pro-preview",
            "temperature": 0.3,
            "top_p": 0.9,
            "max_output_tokens": 8192,
        },
        "compact": {
            "model": "gemini-2.5-flash",
            "temperature": 0.2,
            "top_p": 0.85,
            "max_output_tokens": 3000,
        }
    }
    
    def __init__(self, api_key: str):
        genai.configure(api_key=api_key)
    
    async def stream_analyze(
        self,
        system_prompt: str,
        task_prompt: str,
        mode: str = "standard"
    ) -> AsyncGenerator[str, None]:
        """
        流式生成解析结果，通过 AsyncGenerator 逐块推送文本。
        """
        config = self.MODEL_CONFIGS.get(mode, self.MODEL_CONFIGS["standard"])
        
        model = genai.GenerativeModel(
            model_name=config["model"],
            system_instruction=system_prompt,
            generation_config=GenerationConfig(
                temperature=config["temperature"],
                top_p=config["top_p"],
                max_output_tokens=config["max_output_tokens"],
            )
        )
        
        # 使用线程池运行同步的流式调用
        loop = asyncio.get_event_loop()
        
        def _sync_stream():
            return model.generate_content(task_prompt, stream=True)
        
        response = await loop.run_in_executor(None, _sync_stream)
        
        for chunk in response:
            if chunk.text:
                yield chunk.text
    
    async def analyze(
        self,
        system_prompt: str,
        task_prompt: str,
        mode: str = "standard"
    ) -> str:
        """
        非流式调用，返回完整结果字符串。
        用于格式校验和修复场景。
        """
        full_text = ""
        async for chunk in self.stream_analyze(system_prompt, task_prompt, mode):
            full_text += chunk
        return full_text
```

### 6.3 响应解析与格式校验器（OutputFormatter）

```python
# output_formatter.py

import re
from dataclasses import dataclass
from typing import Optional


@dataclass
class ValidationResult:
    is_valid: bool
    issues: list[str]
    fixed_output: Optional[str] = None


class OutputFormatter:
    """
    验证 Gemini 输出是否符合 VitePress Tabs 格式规范，
    并尝试自动修复常见格式问题。
    """
    
    REQUIRED_TABS = [
        "== 完整翻译",
        "== 语义/意群拆分",
        "== 关键词汇与短语",
        "== 重难点与语法分析",
        "== 导师小结"
    ]
    
    def validate(self, output: str) -> ValidationResult:
        issues = []
        
        # 检查 <details> 包裹
        if not output.strip().startswith("<details>"):
            issues.append("缺少 <details> 开始标签")
        if not output.strip().endswith("</details>"):
            issues.append("缺少 </details> 结束标签")
        
        # 检查 <summary> 标签
        if "<summary>" not in output or "</summary>" not in output:
            issues.append("缺少 <summary> 标签")
        
        # 检查 ::: tabs 语法
        if "::: tabs" not in output:
            issues.append("缺少 ::: tabs 声明")
        if ":::" not in output[output.rfind("::: tabs"):]:
            issues.append("缺少 ::: 结束标记")
        
        # 检查五大标签页
        for tab in self.REQUIRED_TABS:
            if tab not in output:
                issues.append(f"缺少标签页：{tab}")
        
        # 检查 Emoji（Unicode 范围检测）
        emoji_pattern = re.compile(
            "[\U0001F600-\U0001F64F\U0001F300-\U0001F5FF"
            "\U0001F680-\U0001F6FF\U0001F1E0-\U0001F1FF]+"
        )
        if emoji_pattern.search(output):
            issues.append("输出中包含 Emoji 字符")
        
        if not issues:
            return ValidationResult(is_valid=True, issues=[])
        
        # 尝试自动修复
        fixed = self._attempt_fix(output, issues)
        return ValidationResult(is_valid=False, issues=issues, fixed_output=fixed)
    
    def _attempt_fix(self, output: str, issues: list[str]) -> Optional[str]:
        fixed = output.strip()
        
        # 修复首尾标签
        if not fixed.startswith("<details>"):
            fixed = "<details>\n<summary>句子解析</summary>\n" + fixed
        if not fixed.endswith("</details>"):
            fixed = fixed + "\n</details>"
        
        # 移除 Emoji
        emoji_pattern = re.compile(
            "[\U0001F600-\U0001F64F\U0001F300-\U0001F5FF"
            "\U0001F680-\U0001F6FF\U0001F1E0-\U0001F1FF]+"
        )
        fixed = emoji_pattern.sub("", fixed)
        
        return fixed
```

```python
# main.py

from fastapi import FastAPI, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from sse_starlette.sse import EventSourceResponse
from pydantic import BaseModel
import asyncio
import json

from prompt_engine import PromptEngine, AnalysisRequest
from gemini_client import GeminiClient
from output_formatter import OutputFormatter
from database import HistoryDB
from utils import extract_trunk, detect_language

app = FastAPI(title="英语长难句解析系统", version="1.0.0")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_methods=["*"],
    allow_headers=["*"],
)

# 依赖注入
prompt_engine = PromptEngine()
gemini_client = GeminiClient(api_key="YOUR_GEMINI_API_KEY")
formatter = OutputFormatter()
db = HistoryDB()


class AnalyzeRequest(BaseModel):
    sentence: str
    mode: str = "standard"
    model: str = "standard"
    context: str | None = None


@app.post("/analyze")
async def analyze_sentence(request: AnalyzeRequest):
    """
    主解析接口，返回 SSE 流式响应。
    """
    # 输入验证
    if not request.sentence.strip():
        raise HTTPException(status_code=400, detail="输入句子不能为空")
    
    if detect_language(request.sentence) != "en":
        raise HTTPException(status_code=400, detail="请输入英文句子")
    
    # 提取主干
    trunk = extract_trunk(request.sentence)
    
    # 构造解析请求
    analysis_request = AnalysisRequest(
        sentence=request.sentence,
        trunk_text=trunk,
        mode=request.mode,
        context_sentences=request.context
    )
    
    # 构造提示词
    task_prompt = prompt_engine.build_task_prompt(analysis_request)
    
    async def event_generator():
        full_output = ""
        
        yield {"event": "start", "data": json.dumps({"trunk": trunk})}
        
        try:
            async for chunk in gemini_client.stream_analyze(
                system_prompt=prompt_engine.system_prompt,
                task_prompt=task_prompt,
                mode=request.model
            ):
                full_output += chunk
                yield {"event": "chunk", "data": chunk}
            
            # 格式验证
            validation = formatter.validate(full_output)
            if not validation.is_valid and validation.fixed_output:
                full_output = validation.fixed_output
                yield {"event": "fixed", "data": json.dumps({
                    "issues": validation.issues
                })}
            
            # 持久化
            record_id = await db.save(
                sentence=request.sentence,
                output=full_output,
                mode=request.mode
            )
            
            yield {"event": "done", "data": json.dumps({
                "record_id": record_id,
                "is_valid": validation.is_valid
            })}
            
        except Exception as e:
            yield {"event": "error", "data": json.dumps({"message": str(e)})}
    
    return EventSourceResponse(event_generator())


@app.get("/history")
async def get_history(page: int = 1, page_size: int = 20):
    return await db.list_records(page=page, page_size=page_size)


@app.delete("/history/{record_id}")
async def delete_record(record_id: int):
    await db.delete_record(record_id)
    return {"status": "deleted"}
```

### 7.1 前端 Pinia Store 设计

```javascript
// stores/analyzer.js

import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import axios from 'axios'

export const useAnalyzerStore = defineStore('analyzer', () => {
  // 状态
  const inputSentence = ref('')
  const outputMarkdown = ref('')
  const isStreaming = ref(false)
  const streamProgress = ref(0)
  const currentRecordId = ref(null)
  const errorMessage = ref('')
  const selectedMode = ref('standard')
  const selectedModel = ref('standard')
  
  // 历史记录
  const history = ref([])
  const historyPage = ref(1)
  const historyTotal = ref(0)
  
  // 计算属性
  const hasOutput = computed(() => outputMarkdown.value.length > 0)
  const canAnalyze = computed(
    () => inputSentence.value.trim().length > 0 && !isStreaming.value
  )
  
  // 核心解析方法
  async function analyze() {
    if (!canAnalyze.value) return
    
    outputMarkdown.value = ''
    isStreaming.value = true
    errorMessage.value = ''
    
    try {
      const response = await fetch('/api/analyze', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          sentence: inputSentence.value,
          mode: selectedMode.value,
          model: selectedModel.value
        })
      })
      
      const reader = response.body.getReader()
      const decoder = new TextDecoder()
      
      while (true) {
        const { done, value } = await reader.read()
        if (done) break
        
        const text = decoder.decode(value)
        const lines = text.split('\n')
        
        for (const line of lines) {
          if (line.startsWith('event: chunk')) continue
          if (line.startsWith('data: ')) {
            const data = line.slice(6)
            // 解析 SSE 事件
            try {
              const json = JSON.parse(data)
              if (json.record_id) {
                currentRecordId.value = json.record_id
                isStreaming.value = false
              }
            } catch {
              // 普通文本块，直接追加
              outputMarkdown.value += data
            }
          }
        }
      }
    } catch (err) {
      errorMessage.value = err.message
      isStreaming.value = false
    }
  }
  
  async function loadHistory(page = 1) {
    const res = await axios.get(`/api/history?page=${page}&page_size=20`)
    history.value = res.data.records
    historyTotal.value = res.data.total
    historyPage.value = page
  }
  
  function loadRecord(record) {
    inputSentence.value = record.sentence
    outputMarkdown.value = record.output
    currentRecordId.value = record.id
  }
  
  return {
    inputSentence, outputMarkdown, isStreaming, streamProgress,
    currentRecordId, errorMessage, selectedMode, selectedModel,
    history, historyPage, historyTotal,
    hasOutput, canAnalyze,
    analyze, loadHistory, loadRecord
  }
})
```

### 7.2 Markdown 渲染配置

```javascript
// utils/markdown-renderer.js

import MarkdownIt from 'markdown-it'
import MarkdownItContainer from 'markdown-it-container'
import hljs from 'highlight.js'

const md = new MarkdownIt({
  html: true,          // 允许 HTML（用于 <details> <summary>）
  linkify: true,
  typographer: true,
  highlight: (str, lang) => {
    if (lang && hljs.getLanguage(lang)) {
      return hljs.highlight(str, { language: lang }).value
    }
    return ''
  }
})

// 注册 ::: tabs 容器
md.use(MarkdownItContainer, 'tabs', {
  render(tokens, idx) {
    if (tokens[idx].nesting === 1) {
      return '<div class="tabs-container"><div class="tabs-nav"></div><div class="tabs-content">'
    }
    return '</div></div>'
  }
})

export function renderMarkdown(text) {
  // 彻底移除无意义的 <details>/<summary> 折叠，让分析结果平铺直观呈现
  let processedText = text;
  processedText = processedText.replace(/<details[^>]*>/g, '');
  processedText = processedText.replace(/<\/details>/g, '');
  processedText = processedText.replace(/<summary>(.*?)<\/summary>/g, '');
  return md.render(processedText)
}
```
